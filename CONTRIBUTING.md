## ğŸ“‹ Contribution Guidelines

### Code Style
- Use TypeScript for new files
- Follow existing code formatting
- Add JSDoc comments for functions
- Include types for all parameters

### Testing
- Add tests for new features
- Ensure existing tests pass
- Test with actual API calls (when possible)
- Include example outputs in PR

### Documentation
- Update README.md if adding features
- Add inline comments for complex logic
- Update setup guides for configuration changes
- Include screenshots for UI changes

### Commit Messages
- Use present tense: "Add feature" not "Added feature"
- Be descriptive: "Add ElevenLabs voice cloning" not "Update code"
- Reference issues: "Fix #123: Audio sync issue"

## ğŸ› Bug Reports
